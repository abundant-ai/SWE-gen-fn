Swagger/OpenAPI output generated by PostgREST is missing the ability to specify upsert behavior for POST requests via the `Prefer` header. As a result, when users open the generated spec in Swagger UI and try to insert rows, there is no documented header option to send `Prefer: resolution=merge-duplicates`, so they cannot perform upserts from Swagger UI even though PostgREST supports it.

When requesting the OpenAPI document (for example by sending `Accept: application/openapi+json` to the root endpoint), the OpenAPI operation object for table POST endpoints should include header parameters that expose the relevant `Prefer` options for upserts. In particular, POST operations should document an optional `Prefer` header that allows choosing `resolution=merge-duplicates` (and the related upsert resolution behavior), so Swagger UI renders a dropdown/input for that header.

Currently, POST endpoints in the OpenAPI output do not include this upsert-related `Prefer` option. Update OpenAPI generation so that for table insert (POST) operations, the OpenAPI `parameters` include the appropriate `Prefer` header parameter(s) that cover upsert resolution, enabling Swagger UI to perform upserts by setting `Prefer: resolution=merge-duplicates` on POST requests.

Expected behavior: the OpenAPI JSON for a table POST path includes header parameter definitions for `Prefer` that include the upsert resolution option `resolution=merge-duplicates`, making it selectable/enterable in Swagger UI.

Actual behavior: the OpenAPI JSON for table POST paths lacks any `Prefer` parameter option for upsert resolution, so Swagger UI cannot be used to trigger upserts.
