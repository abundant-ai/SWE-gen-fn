ColocatedJS/ColocatedHooks currently cannot import third-party JavaScript dependencies that are installed in the standard Phoenix project layout under the assets directory (for example, packages installed via npm into assets/node_modules). When a colocated script contains an import like:

```js
import flatpickr from "flatpickr";
```

the JavaScript build step fails with an error like:

"Could not resolve \"flatpickr\"" and the build exits non-zero.

This happens in the default project configuration generated by Phoenix, where JS dependencies live under assets/node_modules. As a result, colocated JS compilation/bundling resolves imports only from other locations (such as deps/build output), but not from the app’s assets/node_modules, making common package imports unusable in colocated scripts.

Update ColocatedJS so that, by default, package resolution includes the project’s assets/node_modules without requiring users to manually change their build tool environment. After the fix, colocated scripts and hooks should be able to import packages installed in assets/node_modules using bare specifiers (e.g., "flatpickr", "dayjs") and the build should succeed.

Additionally, document the intended/recommended node_modules handling for ColocatedJS so that users understand where to install JavaScript dependencies (assets vs project root) and why imports work in the supported setup.
