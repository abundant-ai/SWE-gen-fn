Generating a new Phoenix project (including umbrella projects) produces code that uses an outdated Gettext API. After updating Phoenix/Gettext, freshly generated projects should compile and run using the current Gettext API, but the generator output still references the old API and/or produces inconsistent Gettext-related modules/usages.

When running `Mix.Tasks.Phx.New.run/1` to generate a new application with defaults, the generated project should include Gettext usage that matches the new Gettext API across all generated modules and templates (including web modules and any generated components). This should also hold when running `Mix.Tasks.Phx.New.run/1` with `--umbrella`, where both the “app” and “web” applications are generated.

The problem shows up as generator output being out of sync with the updated Gettext API, leading to compilation errors in the generated project when it calls or imports Gettext functionality using the old interface. Update the Phoenix generators so that the generated code consistently uses the new Gettext API everywhere it appears, for both standard and umbrella project layouts, and ensure the installer templates remain consistent with the canonical Phoenix priv/template sources used by the generators.

After the fix, generating a project with `phx.new` should produce a project that (1) compiles without Gettext-related errors, (2) has consistent Gettext module definitions and call sites aligned with the new API, and (3) produces the same expected static/template content that Phoenix ships (for example, shared assets/components templates must remain in sync between the generator’s templates and Phoenix’s own canonical copies).
