When a spawned domain terminates with an uncaught exception, calling `Domain.join d` should re-raise that exception in the joining domain while preserving the original backtrace from the terminated domain. Currently, `Domain.join` re-raises the exception but loses the originating domain’s backtrace, so `Printexc.get_raw_backtrace ()` (or printing the backtrace) after catching the exception from `Domain.join` shows an incomplete or incorrect trace that does not include the call stack from where the exception was raised inside the spawned domain.

Reproduction: create mutually recursive functions (e.g., `foo` and `bar`) that call each other repeatedly and eventually raise via `failwith`, then run them in a separate domain using `Domain.spawn` and immediately `Domain.join`. Catch the exception around `Domain.join` and print the raw backtrace. The expected output includes frames from the raising function (e.g., `Stdlib.failwith`) and the chain of calls inside the spawned domain (`foo`/`bar` recursion), followed by frames showing the exception being re-raised in the domain runtime (`Stdlib__Domain.spawn.body` and `Stdlib__Domain.join`).

Expected behavior: the backtrace printed/observed in the joining domain after `Domain.join` raises must match the spawned domain’s backtrace (including the internal call stack leading to `failwith`), and it should also show that it was re-raised at `Stdlib__Domain.join`.

Actual behavior: the exception is raised, but the backtrace lacks the spawned domain’s call stack (often only showing frames in `Domain.join` or other caller frames), making debugging of failures in spawned domains difficult.

Implement support so that `Domain.join` captures the terminating domain’s backtrace at the point of uncaught exception and restores it when re-raising in the joining domain, so that standard backtrace facilities (`Printexc.get_raw_backtrace`, `Printexc.print_raw_backtrace`, and `-g`/`ocamlrunparam b=1`) report the correct originating trace.
